# Project-BD
Создаю проект по базам данных включающий в себя ER-диаграмму и работу с PostqreSQL.

Создал ER-диаграмму онлайн магазина спортивной одежды и создал таблицы в PostgreSQL и заполнил их данными
# Структура базы даных 
у меня получилось 10  таблиц

# Таблицы:
| Таблица | Расшифровка |
|--------- | ---------- |
|`users` | пользователи|
|`categories` | категории|
|`order-items` | количество товара в заказе|
|`brands` | бренды |
|`products` | продукты|
|`warehouses` | склад|
|`stock` | остатки |
|`payments` | оплата|
|`delivery` | доставка|
|`orders` | заказы|

# Связи
- **categories** → **products** (один ко многим)
- **brands** → **products** (один ко многим)
- **users** → **orders** (один ко многим)
- **orders** → **order_items** (один ко многим)
- **products** → **order_items** (один ко многим)
- **warehouses** → **stock** (один ко многим)
- **products** → **stock** (один ко многим) → вместе образуют M:N
- **orders** → **payments** (один к одному)
- **orders** → **delivery** (один к одному)
- **products** ↔ **warehouses** (многие ко многим) → через stock
# Заполнение 
Я заполнил свои таблицы случайными данными
- 3 пользователя с полными ФИО, email, телефонами и датой регистрации
- 5 категорий товаров (MMA, фитнес, бег, йога, питание)
- 5 брендов из разных стран с годом основания
- 8 продуктов с артикулами, описаниями, ценами и указанием оригинальности
- 3 склада с названиями и адресами
- 10 записей о наличии товаров на складах (stock)
- 3 способа оплаты с разными статусами
- 3 варианта доставки (курьер, самовывоз, почта)
- 3 заказа с уникальными номерами, статусами, суммами и привязкой к пользователям, оплатам и доставкам
- 4 позиции заказов (order_items) — по 1–2 товара в каждом заказе
